About Xenqtt

The Xenqtt initiative was started because we realized that there were certain gaps in how enterprise architectures and legacy systems could integrate with an M2M ecosystem communicating via MQTT. These gaps were:

  • Clustered applications cannot act as a single MQTT client. This could lead to scaling issues or duplicate message delivery
  • Applications using HTTP exclusively had no way to participate in this new ecosystem

Xenqtt addresses these gaps by providing a proxy for multi-client management with integration to an MQTT system and a gateway that facilitates the translation of HTTP messages into MQTT and vice-versa. Additionally, Xenqtt provides a fully-featured MQTT client that applications can use to communicate via the MQTT protcool. This helps to make Xenqtt a one-stop shop for all your MQTT needs.
